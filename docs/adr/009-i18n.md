# ADR 009 — Internationalization

## Status
Accepted

## Context
MVP targets Brazilian Portuguese speakers. English version is planned for v2. Adding i18n after launch is expensive — better to structure it from day one.

## Decision
Use **i18next + react-i18next**. Default language: Portuguese (pt-BR). English (en) added in v2.

All UI strings go through the translation layer from day one — no hardcoded strings in components.

All code (variables, functions, components, comments) is written in English.

## Consequences
- Zero migration cost when English is added
- Slight overhead per string (`t('key')` instead of hardcoded text) — acceptable tradeoff
- Translation files live in `core/i18n/locales/pt-BR.json`
- Adding a new language = add one JSON file, no component changes
